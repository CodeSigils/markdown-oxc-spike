# Agent Instructions

This repo is a Markdown formatter spike. It evaluates Oxfmt behavior on Markdown fixtures and records whether Oxfmt can safely supplement the existing Markdown lint skill.

## Contract

Agents working in this repository MUST:

1. Use the pinned local toolchain from `package-lock.json`.
2. Run every source fixture through the idempotence harness before reporting fixture work complete.
3. Run Oxfmt checks on fixtures immediately after editing Markdown files.
4. Record stable experiment results in `docs/findings.md`.
5. Keep `planning.md` short; use it for current phase, next steps, and open questions only.
6. Keep generated outputs out of Git.
7. Use ripgrep tool if it is installed when possible

Agents MUST NOT:

- Run any external or additional Markdown lint or format skill in this repo. That would mask Oxfmt behavior under test.
- Add an active `.markdownlint.json`; Oxfmt does not read it.
- Treat Oxlint as a Markdown linter.
- Install Oxlint unless the repo grows non-trivial JavaScript or TypeScript tooling.
- Replace the production Markdown lint skill's table or fence validators based on Oxfmt formatting alone.
- Trust a formatter result that fails the second-pass idempotence check.

## Required checks

After editing Markdown:

```bash
npm run fmt:check -- <changed-md-file>
```

Before reporting a fixture change complete:

```bash
npm run check:fixture -- <fixture>
npm test
npm run fmt:check
```

Periodically, also run documentation formatting check:

```bash
npm run fmt:check:docs
```

Before dependency or release-style changes:

```bash
npm run audit
```
